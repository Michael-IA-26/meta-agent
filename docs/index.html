Email Agent

Pipeline quotidien — Gmail → Claude → Rapport HTML → Telegram. Tourne à 08h45 chaque matin.

L'Email Agent récupère automatiquement les emails non lus depuis Gmail via OAuth2, les analyse un par un avec Claude Sonnet 4 en tenant compte du profil ICP (Ideal Customer Profile), classifie chaque email (prospect chaud / froid / hors-cible / question technique), persiste les résultats dans Supabase, génère un rapport HTML responsive, l'envoie par email, puis pousse un résumé Telegram avec les KPIs de la session. Tout se fait sans intervention humaine.

Les 6 agents

📥

gmail_fetcher

1 · Fetch

Se connecte à Gmail via OAuth2, récupère les N derniers emails non lus, les normalise en dict structuré.

Input
  • max_results int (défaut 20)
  • Token OAuth2 base64
Output
  • list[EmailRaw]
  • from, subject, body, date
  • message_id (dedup)
📋 — aucune table (fetch only)
🧠

email_analyzer

2 · Analyze

Envoie chaque email à Claude API avec le contexte ICP. Classifie l'intent, extrait les infos clés, propose une réponse.

Input
  • EmailRaw
  • IcpContext (chargé 1×)
Output
  • intent: prospect chaud/froid
  • priority, summary
  • action, suggested_reply
🤖 Claude Sonnet 4 API
💾

supabase_writer

3 · Persist

Persiste chaque email analysé dans Supabase. Gère les doublons par message_id. Calcule et sauvegarde les KPIs.

Input
  • EmailAnalyzed
  • ou KpiPayload
Output
  • bool succès
  • ou KpiResult
🗄️ emails 🗄️ email_kpis
📝

reporter

4 · Build

Agrège les résultats de la session. Génère un rapport HTML responsive avec statistiques, tableau par catégorie et graphiques.

Input
  • list[EmailAnalyzed]
  • elapsed_sec
Output
  • str HTML complet
  • stats, tableaux, graphiques
📋 — aucun appel réseau
📤

sender

5 · Send

Envoie le rapport HTML par email SMTP à l'adresse configurée. Ne lève jamais d'exception — dégradation gracieuse.

Input
  • html str
  • subject str
  • recipient (env)
Output
  • bool succès
📧 SMTP / Gmail API
💬

telegram_sender

6 · Notify

Envoie une alerte Telegram avec les KPIs de la session : nb emails traités, % prospects chauds, action recommandée.

Input
  • list[EmailAnalyzed]
  • KpiResult
Output
  • bool succès
💬 Telegram Bot API

LeadCommercial

Détection automatique de leads B2B — nouvelles entreprises IDF chaque nuit, scoring ICP, enrichissement, alerte Telegram.

Le pipeline tourne chaque nuit à 02h00 (Europe/Paris). Il interroge l'API INSEE Sirene pour récupérer toutes les nouvelles immatriculations d'Île-de-France (75, 77, 78, 91, 92, 93, 94, 95), score chaque entreprise de 0 à 100 selon les critères ICP configurés (secteur, forme juridique, zone géographique), enrichit les leads qualifiés via Pappers (dirigeant, capital, santé financière), persiste en Supabase avec verrou anti-doublon multi-cabinets, et envoie une alerte Telegram instantanée pour chaque lead score > 70.

Résultat concret : 76% des leads IDF qualifiés en moyenne — 11 alertes Telegram envoyées sur de vraies entreprises en test.

Les 5 agents

🏢

sirene_fetcher

1 · Fetch

Interroge l'API INSEE Sirene chaque nuit pour récupérer toutes les nouvelles immatriculations IDF du jour. Filtre par forme juridique et code NAF.

Input
  • Date (défaut: hier)
  • Départements 75-95
Output
  • list[CompanyRaw]
  • siren, siret, denomination
  • naf, dept, commune, date
🏛️ INSEE Sirene API
🔎

pappers_enricher

2 · Enrich

Enrichit chaque lead avec les données Pappers : dirigeant, capital social, date de création, score santé financière. Dégradation gracieuse si absent.

Input
  • siren (str)
Output
  • dirigeant nom/prenom
  • capital_social, site_web
  • score_sante (0-100)
📊 Pappers API
⚖️

icp_scorer

3 · Score

Note chaque lead de 0 à 100 selon les critères ICP configurés (secteur, zone géographique, forme juridique, taille). Explique le score.

Input
  • CompanyRaw
  • IcpConfig
Output
  • score (0–100)
  • qualified (bool)
  • scoring_details
📋 — logique règles métier
🔒

supabase_storage

4 · Persist

Sauvegarde les leads scorés dans Supabase avec gestion des doublons par SIREN. Met à jour le statut des leads existants.

Input
  • ScoredLead
  • cabinet_id (lock)
Output
  • bool
  • False si SIREN verrouillé
🗄️ leads 🗄️ lead_locks
📲

telegram_notifier

5 · Notify

Envoie les alertes Telegram pour les leads score > 70, avec nom, SIREN, secteur, score et lien Pappers. Silent si tokens absents.

Input
  • ScoredLead
  • tous champs enrichis
Output
  • bool succès
💬 Telegram Bot API

JM Partners — Cabinet Comptable

Assistant comptable complet. 9 agents spécialisés. 3 sprints livrés. Beta le 8 juin 2026.

JM Partners est un cabinet d'expertise comptable accompagnant des TPE/PME (SARL, SAS, EURL). Le système automatise les tâches répétitives à faible valeur ajoutée : relances clients pour documents manquants, surveillance du calendrier fiscal (TVA, IS, bilans), alertes proactives J-15/J-7/J-3, et notifications multi-canal (email + Telegram) selon l'urgence.

Un dashboard FastAPI offre un kanban dossiers (Documents manquants / En attente / Complet), un calendrier échéances J-30/J-15/J-7/J-3, et un bouton dry-run pour tester sans envoi réel. APScheduler exécute un cycle complet toutes les 30 minutes en production.
