Email Agent

Pipeline quotidien de traitement des emails par IA — analyse, classement, rapport & alertes.

L'Email Agent tourne chaque matin à 08h45. Il récupère les emails non lus depuis Gmail, les analyse un par un avec Claude Sonnet 4.6 (priorité, catégorie, résumé, action suggérée, brouillon de réponse), persist les résultats dans Supabase, génère un rapport HTML envoyé par email, puis pousse un résumé Telegram avec les KPIs de la semaine (temps gagné, valeur estimée).

Les 6 agents

📥

GmailFetcher

1 · Fetch

Récupère les emails non lus via l'API Gmail (OAuth prod base64 ou token local).

Input
  • max_results int (défaut 20)
Output
  • list[EmailRaw]
  • id, subject, from, date, body
🧠

EmailAnalyzer

2 · Analyze

Classifie chaque email avec Claude Sonnet 4.6 et le contexte ICP du cabinet.

Input
  • EmailRaw
  • IcpContext (chargé 1×)
Output
  • EmailAnalyzed
  • priority, category, summary
  • action, suggested_reply
💾

SupabaseWriter

3 & 6 · Persist

Sauvegarde les emails analysés et les KPIs hebdomadaires dans Supabase (2 appels).

Input
  • EmailAnalyzed
  • ou list[EmailAnalyzed] + elapsed_sec
Output
  • bool (succès)
  • ou KpiResult
📝

ReportBuilder

4 · Build

Génère un rapport HTML structuré par priorité (aucun appel réseau).

Input
  • list[EmailAnalyzed]
Output
  • str HTML
  • organisé par priorité
📤

GmailReporter

5 · Send

Envoie le rapport HTML par email via l'API Gmail. Ne lève jamais d'exception.

Input
  • html str
  • subject str
  • recipient? (défaut: RAPPORT_EMAIL)
Output
  • bool (succès)
💬

TelegramSender

7 · Notify

Envoie un résumé Markdown sur Telegram avec KPIs (temps gagné, valeur estimée).

Input
  • list[EmailAnalyzed]
  • KpiResult?
Output
  • bool (succès)

LeadCommercial

Détection automatique de leads B2B qualifiés — nouvelles entreprises IDF, scoring ICP, enrichissement Pappers, alerte Telegram.

Le pipeline tourne chaque nuit à 02h00 (Europe/Paris). Il récupère les établissements nouvellement créés en Île-de-France via l'API Sirene (INSEE), score chaque entreprise selon des règles ICP (score 0–100), enrichit les leads qualifiés via l'API Pappers (dirigeant, site web, capital), persiste en Supabase avec un mécanisme de verrou anti-doublon multi-cabinets, et envoie une alerte Telegram instantanée. Un flag --dry-run permet de tester sans écriture.

Les 5 agents

🏢

SireneFetcher

1 · Fetch

Interroge l'API Sirene (INSEE) pour récupérer les entreprises IDF créées récemment.

Input
  • SireneInput
  • max_results, date
Output
  • list[CompanyRaw]
  • siren, siret, denomination
  • naf, dept, commune, date
⚖️

LeadScorer

2 · Score

Calcule un score 0–100 basé sur des règles de signaux (création, fiscal, intention…) et l'ICP cabinet.

Input
  • ScoreInput
  • company, signal_type, icp?
Output
  • ScoredLead
  • score (0–100)
  • qualified (bool)
  • scoring_details
🔎

PappersEnricher

3 · Enrich

Enrichit le lead via l'API Pappers : dirigeant, site web, capital social. Dégradation gracieuse si absent.

Input
  • EnrichInput
  • siren
Output
  • PappersEnrichment
  • dirigeant nom/prenom/email
  • site_web, capital_social
🔒

SupabaseWriter

4 · Persist

Vérifie le verrou anti-doublon (lead_locks), puis insère le lead dans la table leads.

Input
  • WriteInput
  • siren + 20+ champs
Output
  • bool
  • False si SIREN déjà verrouillé
📲

TelegramNotifier

5 · Notify

Envoie une alerte Telegram Markdown pour chaque lead qualifié persisté. Silent si tokens absents.

Input
  • NotifyInput
  • tous les champs du lead
Output
  • bool (succès)
